# Лабораторна робота №2: Основні інструкції 32-бітного ARM процесора для мікроконтролерів

Цей репозиторій містить результати виконання другої лабораторної роботи з вивчення та використання асемблерних інструкцій ядра Cortex-M4 на прикладі розробки власної процедури для мікроконтролера STM32F407VG.

## 🎯 Мета проекту

Навчитися використовувати асемблерні інструкції ядра Cortex-M4, працювати з процедурами та розуміти синтаксис мови асемблера GAS (GNU Assembly). Закріпити навички аналізу роботи регістрів за допомогою відлагоджувача GDB.

## 📂 Структура репозиторію

* `lab1.S` — головний файл із реалізацією логіки програми за варіантом №4: обчислення функції `{(a|b)*c, якщо (a*b)>=10; (a*b)*c, якщо (a*b)<10}` з використанням математичних інструкцій та блоків умовного виконання (ITE).
* `start.S` — оновлений вихідний файл ініціалізації, який містить налаштування вказівника стеку, таблицю векторів виключень та виклик основної процедури нашої програми.
* `lscript.ld` — скрипт лінкера, що визначає мапу пам'яті мікроконтролера (розподіл секцій FLASH та RAM).
* `Makefile` — конфігураційний файл для автоматизації процесів крос-компіляції (включаючи нові об'єктні файли), компонування та запуску емулятора.

## 🛠 Вимоги до програмного забезпечення

Для успішної збірки та запуску проекту в середовищі Linux необхідно мати встановлені наступні інструменти:

* **GNU Arm Embedded Toolchain** (`arm-none-eabi-gcc`, `arm-none-eabi-binutils`, `gdb-multiarch`).
* **Емулятор QEMU** (рекомендовано xPack QEMU Arm версії 7.2.0 або сумісний gnuarmeclipse).
* **Утиліта GNU Make**.

## 🚀 Інструкція з запуску та відлагодження

**1. Збірка проекту**

Для компіляції вихідного коду та створення бінарного файлу прошивки (`firmware.bin`) виконайте в терміналі команду:

```bash
make
```

**2. Запуск емулятора**

Щоб запустити віртуальну плату STM32F4-Discovery та відкрити порт `tcp::1234` для відлагоджувача, виконайте:

```bash
make qemu
```

*Примітка: термінал перейде в режим очікування з'єднання.*

**3. Підключення GDB та покрокове виконання**

Відкрийте нове вікно терміналу та запустіть відлагоджувач:

```bash
gdb-multiarch firmware.elf
```

У консолі GDB підключіться до емулятора:

```gdb
target extended-remote :1234
```

Для зручного відслідковування змін у регістрах процесора під час виконання арифметичних операцій увімкніть графічний режим відображення:

```gdb
layout regs
```

Для покрокового виконання інструкцій використовуйте команду `step` (або `Enter`).

## 🧹 Очищення робочої директорії

Щоб видалити всі згенеровані об'єктні та бінарні файли (`.o`, `.elf`, `.bin`), використовуйте:

```bash
make clean
```
